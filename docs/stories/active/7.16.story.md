## Story

**As a** JARVIS user,
**I want** JARVIS to generate formatted reports and export data as files I can download,
**so that** I can request analysis, summaries, or data and receive them as ready-to-use documents.

## Acceptance Criteria

1. New tool `generate_report` creates markdown reports and converts to HTML for browser viewing
2. New tool `export_data` creates downloadable files: CSV, JSON, or plain text
3. Reports are served via `GET /api/brain/reports/{id}` with a shareable link
4. Cockpit shows a download button when a report/export is generated
5. Reports include: title, date, sections with headers, tables, and optional charts (markdown tables)
6. CSV export: proper escaping, UTF-8 BOM for Excel compatibility
7. Reports are stored in `data/reports/` with auto-cleanup after 7 days
8. Voice command support: "gera um relatorio de...", "exporta os dados de..."

## Dev Notes

### Architecture Context

[Source: apps/jarvis-hub/server.js, cloud-brain.js]

Reports sao uma extensao natural dos tools existentes. O agente ja pode ler arquivos e gerar conteudo — esta story adiciona a capacidade de formatar e servir como documentos downloadaveis.

### Markdown → HTML

Usar `marked` (ja popular, leve) para converter markdown em HTML. Adicionar CSS inline basico para que o HTML seja apresentavel standalone.

```javascript
const marked = require('marked');
const html = `<!DOCTYPE html>
<html><head><meta charset="utf-8">
<style>body{font-family:system-ui;max-width:800px;margin:0 auto;padding:2rem}
table{border-collapse:collapse;width:100%}td,th{border:1px solid #ddd;padding:8px}
</style></head><body>${marked.parse(markdown)}</body></html>`;
```

### CSV UTF-8 BOM

Excel precisa do BOM para interpretar UTF-8 corretamente:
```javascript
const bom = '\uFEFF';
const csv = bom + headers.join(',') + '\n' + rows.map(r => r.join(',')).join('\n');
```

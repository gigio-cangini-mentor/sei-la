## Story

**As a** JARVIS Cloud Brain operator,
**I want** a complete audit log of all agent actions,
**so that** I can review what happened, debug issues, and detect suspicious activity.

## Acceptance Criteria

1. Every tool execution is logged with: timestamp, agent ID, tool name, arguments (sanitized), result status, duration, user session
2. Every agent invocation is logged with: timestamp, agent ID, command, model used, token usage, cost estimate
3. Audit logs are written to `data/audit/YYYY-MM-DD.jsonl` (one file per day, JSONL format)
4. Sensitive data is redacted from logs: API keys, passwords, file contents over 500 chars (replaced with `[REDACTED:N chars]`)
5. A REST endpoint `GET /api/brain/audit` returns recent audit entries (last 100, paginated, filterable by agent/tool/date)
6. Audit logs rotate: files older than 30 days are automatically compressed (gzip), files older than 90 days are deleted
7. Failed/blocked operations are logged with reason (rate limit, sandbox block, authority violation)
8. WebSocket broadcasts audit events in real-time for the cockpit Activity tab

  - [ ] Log blocked operations from rate limiter and sandbox
- [ ] Task 3: REST endpoint (AC: 5)
  - [ ] `GET /api/brain/audit?limit=100&offset=0&agent=dev&tool=git_push&date=2026-02-27`
  - [ ] Read from JSONL files, filter, paginate
  - [ ] Protected by existing PIN auth

  - [ ] On startup and daily interval: compress files > 30 days

  - [ ] Throttle: max 10 events/second to WS clients

## Dev Notes

### Architecture Context

[Source: apps/jarvis-hub/server.js — broadcast(), Activity tab W4.13]

O cockpit ja tem uma Activity tab (W4.13) que mostra timeline de agentes. Esta story adiciona o backend formal de audit logging que alimenta essa tab com dados persistentes.

### Storage Estimate

Cada entry ~500 bytes. Com ~1000 tool executions/dia = ~500KB/dia. 30 dias = ~15MB. Compressao gzip ~85% = ~2.25MB por 30 dias. Muito leve.

## Story

**As a** JARVIS operator,
**I want** Claude Code running on the VPS and accessible through the cockpit as a web terminal,
**so that** I have full Claude Code autonomy from any device — mobile or desktop — through the same cockpit interface.

1. Claude Code CLI is installed on the VPS and functional against `/opt/aios-core/`
2. `ttyd` serves Claude Code as a web terminal on an internal port (not exposed directly)
3. Traefik routes `/terminal` path to ttyd with SSL (same domain: `jarvis.srv1266496.hstgr.cloud/terminal`)
4. ttyd is protected with authentication (reuse cockpit PIN or basic auth)
5. Cockpit has a new "Terminal" tab that embeds the web terminal via iframe
6. Terminal tab is responsive and works on mobile (touch-friendly, proper viewport)
7. VPS timezone is set to `America/Sao_Paulo` (UTC-3)
8. Claude Code session persists across page reloads (tmux/screen backend)
9. Terminal and Cloud Brain Chat operate on the same repo — changes made in one are visible to the other

## Dev Notes

### Architecture Context

[Source: VPS at 72.62.141.8, Traefik reverse proxy, Docker container]

A VPS ja roda Traefik (v3.6.6) como reverse proxy com Let's Encrypt. O JARVIS Hub roda em Docker container. O ttyd pode rodar como servico systemd fora do container (acesso direto ao filesystem) ou dentro de um container separado.

**Recomendacao:** ttyd como systemd service (fora do Docker), pois precisa acessar o filesystem real e o Claude Code instalado globalmente.

### ttyd Configuration

```bash
# /opt/jarvis-hub/scripts/start-terminal.sh
#!/bin/bash
SESSION_NAME="claude-code"
WORK_DIR="/opt/aios-core"

# Create or attach tmux session
tmux has-session -t $SESSION_NAME 2>/dev/null
if [ $? != 0 ]; then
  tmux new-session -d -s $SESSION_NAME -c $WORK_DIR "claude"
fi

# Start ttyd
exec ttyd -p 7681 -W -t fontSize=14 -t theme='{"background":"#1a1a2e"}' tmux attach -t $SESSION_NAME
```

### Traefik Labels (docker-compose)

Se rodar ttyd em container:
```yaml
labels:
  - "traefik.http.routers.terminal.rule=Host(`jarvis.srv1266496.hstgr.cloud`) && PathPrefix(`/terminal`)"
  - "traefik.http.routers.terminal.tls.certresolver=letsencrypt"
  - "traefik.http.services.terminal.loadbalancer.server.port=7681"
```

Se systemd: configurar Traefik file provider para rotear ao localhost:7681.

### Security

- ttyd com `-c user:password` para basic auth
- Ou: Traefik basicauth middleware
- WebSocket upgrade deve funcionar via Traefik (ja funciona para o cockpit WS)

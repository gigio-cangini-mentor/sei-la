## Story

**As a** AIOS developer,
**I want** comprehensive unit tests for all Cloud Brain modules,
**so that** regressions are caught early and the system can be refactored with confidence.

## Acceptance Criteria

1. `orchestrate()` has tests covering: intent detection, agent routing, multi-step detection, error handling
2. `executeAgent()` has tests covering: agentic loop, tool calling, max rounds limit, authority enforcement
3. `executeTool()` has tests covering: each tool type, blocked patterns, path validation, timeout handling
4. `buildSystemPrompt()` has tests covering: persona injection, project knowledge, memory, active story
5. Helper functions tested: `parseAgentYaml()`, `detectAgent()`, `getToolsForAgent()`, `loadAgentMemory()`
6. Test coverage for cloud-brain.js reaches minimum 80% line coverage
7. All tests run without network calls (mock Anthropic API responses)
8. Tests execute in under 30 seconds total

## Dev Notes

### Architecture Context

[Source: apps/jarvis-hub/cloud-brain.js — ~1400 lines]

O cloud-brain.js e um modulo grande (~1400 linhas) sem testes. Todas as funcoes sao exportadas via `module.exports` no final do arquivo. Mocking da Anthropic API e essencial — cada chamada `client.messages.create()` deve ser mockada.

### Mock Strategy

```javascript
jest.mock('@anthropic-ai/sdk', () => ({
  __esModule: true,
  default: jest.fn().mockImplementation(() => ({
    messages: {
      create: jest.fn().mockResolvedValue({
        content: [{ type: 'text', text: '...' }],
        usage: { input_tokens: 100, output_tokens: 50 }
      })
    }
  }))
}));
```

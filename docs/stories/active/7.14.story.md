## Story

**As a** JARVIS user,
**I want** JARVIS to search the web, read news, and access real-time information,
**so that** I can ask anything and get current, accurate answers without leaving the cockpit.

## Acceptance Criteria

1. New tool `web_search` available to all agents: searches the web and returns top results with snippets
2. New tool `web_read` available to all agents: fetches a URL and extracts readable content (article text, not raw HTML)
3. JARVIS can answer questions like "o que esta acontecendo no mercado hoje?", "ultimas noticias sobre IA"
4. Search results include: title, URL, snippet, date (when available)
5. Web content extraction handles: news articles, blog posts, documentation pages, Wikipedia
6. Rate limit: max 10 searches/min, max 20 page reads/min per session
7. Results are cached for 15 minutes (same query returns cached result)
8. Works via both Chat (Cloud Brain) and voice commands

### Content Extraction

```javascript
// Using node-fetch + @mozilla/readability + jsdom
const { JSDOM } = require('jsdom');
const { Readability } = require('@mozilla/readability');

async function extractContent(url) {
  const response = await fetch(url);
  const html = await response.text();
  const dom = new JSDOM(html, { url });
  const article = new Readability(dom.window.document).parse();
  return article ? { title: article.title, content: article.textContent } : null;
}
```

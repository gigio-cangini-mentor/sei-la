## Story

**As a** JARVIS user,
**I want** JARVIS to read and analyze documents from URLs, PDFs, and spreadsheets,
**so that** I can ask questions about any document and get structured analysis or summaries.

## Acceptance Criteria

1. New tool `read_document` handles: PDF URLs, Google Docs (public), Google Sheets (public/CSV export), raw text URLs
2. PDF reading: download → extract text (pdf-parse), return structured content with page numbers
3. Spreadsheet reading: CSV/XLSX download → parse → return as structured data (headers + rows)
4. Google Docs public links: fetch published HTML → extract text via Readability
5. Google Sheets public links: auto-convert to CSV export URL → parse as spreadsheet
6. Max document size: 5MB download limit, 50,000 chars text output limit
7. Document content is injected into agent context for follow-up questions ("analise este documento", "resuma a planilha")
8. Works via Chat and voice commands

## Tasks / Subtasks

- [ ] Task 1: Document type detection (AC: 1)
  - [ ] Detect type from URL pattern and Content-Type header
  - [ ] Google Docs: `docs.google.com/document/d/{id}`
  - [ ] Google Sheets: `docs.google.com/spreadsheets/d/{id}`
  - [ ] PDF: Content-Type `application/pdf` or `.pdf` extension
  - [ ] CSV: Content-Type `text/csv` or `.csv` extension
  - [ ] XLSX: `.xlsx` extension
  - [ ] Fallback: treat as web page (use web_read from 7.14)
- [ ] Task 2: PDF reader (AC: 2)
  - [ ] Download PDF (respect 5MB limit)
  - [ ] Extract with `pdf-parse` (already a dependency)
  - [ ] Return: `{ type: "pdf", pages, text, word_count }`
  - [ ] Truncate at 50K chars with "[... truncated, N more pages]"
- [ ] Task 3: Spreadsheet reader (AC: 3)
  - [ ] Install `xlsx` package (SheetJS) for XLSX parsing
  - [ ] CSV: parse with built-in or `csv-parse`
  - [ ] Return: `{ type: "spreadsheet", sheets: [{ name, headers, rows, row_count }] }`
  - [ ] Limit: first 500 rows per sheet
  - [ ] Format rows as readable text for LLM context
- [ ] Task 4: Google Docs/Sheets handler (AC: 4, 5)
  - [ ] Google Docs: convert URL to published HTML: `https://docs.google.com/document/d/{id}/pub`
  - [ ] Fetch published HTML → extract with Readability
  - [ ] Google Sheets: convert URL to CSV export: `https://docs.google.com/spreadsheets/d/{id}/export?format=csv`
  - [ ] Parse CSV and return structured data
  - [ ] Handle "not public" error gracefully
- [ ] Task 5: Implement read_document tool (AC: 1, 6, 7)
  - [ ] Add `read_document` to TOOLS in cloud-brain.js
  - [ ] Input: `{ url: string, type_hint?: string }`
  - [ ] Output: structured document content
  - [ ] Size guard: 5MB download, 50K chars output
  - [ ] Cache: 30 min TTL per URL
- [ ] Task 6: Context injection for follow-up (AC: 7, 8)
  - [ ] Store last document content in conversation context
  - [ ] Agent can reference it in follow-up messages
  - [ ] Voice shortcuts: "leia este documento", "analise esta planilha", "resuma o PDF"

### Google Docs/Sheets Strategy

Nao usar Google API (requer OAuth). Em vez disso:
- Docs: URL `/pub` serve HTML publico
- Sheets: URL `/export?format=csv` serve CSV publico
- Se nao for publico: retornar erro claro "Documento nao e publico. Compartilhe o link como 'Qualquer pessoa com o link'."

## Story

**As a** AIOS developer,
**I want** deprecated docs archived, orphaned scripts removed, and ghost directories resolved,
**so that** the codebase is clean and new contributors aren't confused by dead artifacts.

## Acceptance Criteria

1. All deprecated docs in `.aios-core/docs/standards/` are moved to `.aios-core/docs/standards/archive/` (5 files identified in backlog)
2. Ghost directories resolved: `.aios-core/agent-teams/` (remove refs or create), `.aios-core/tools/` (remove if empty), `.aios-core/processes/` (remove refs)
3. Dead migration scripts moved to `.aios-core/development/scripts/archive/` (5 scripts identified in backlog)
4. Orphaned infrastructure scripts either wired to agent tasks or archived (5 scripts: changelog-generator, dashboard-status-writer, cicd-discovery, pr-review-ai, test-discovery)
5. Orphaned SQL templates documented with README or wired to @data-engineer tasks (7 templates)
6. All references to moved/archived files are updated (install-manifest, source-tree, agent definitions)
7. `npx aios-core doctor` still passes after cleanup
8. No broken imports or requires after cleanup

## Story

**As a** AIOS developer,
**I want** end-to-end tests covering the full command pipeline,
**so that** the entire flow from user input to agent response is validated automatically.

## Acceptance Criteria

1. E2E test covers the full REST flow: `POST /api/brain/execute` → orchestrate → agent → tools → response
2. E2E test covers the WebSocket flow: WS connect → auth → send command → receive streaming events → final response
3. E2E test covers mission flow: `POST /api/brain/mission` → multi-step → parallel agents → completion
4. E2E test covers QA loop: `POST /api/brain/qa-loop` → qa review → dev fix → re-review cycle
5. E2E test covers error scenarios: invalid auth, malformed request, agent timeout, tool failure
6. All E2E tests use mocked Anthropic API (no real API calls) but real Express server
7. Tests run in under 60 seconds total
8. CI-compatible: can run in headless mode without VPS dependencies

## Dev Notes

### Architecture Context

[Source: apps/jarvis-hub/server.js — Express + WS endpoints]

O server.js exporta `app` e `server` que podem ser iniciados em modo teste. Usar `supertest(app)` para REST e `ws` client para WebSocket. A Anthropic API deve ser mockada em todos os testes.

### Test Setup Pattern

```javascript
let server, app;
beforeAll(async () => {
  // Mock Anthropic
  jest.mock('@anthropic-ai/sdk');
  // Start server on random port
  const module = require('../../apps/jarvis-hub/server');
  // ... setup
});
afterAll(() => server.close());
```

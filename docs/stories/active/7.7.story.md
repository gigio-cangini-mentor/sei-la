## Story

**As a** JARVIS Cloud Brain user,
**I want** the system to recover gracefully from errors and reconnect automatically,
**so that** temporary failures don't require manual intervention.

## Acceptance Criteria

1. Anthropic API errors trigger automatic retry with exponential backoff (max 3 retries, starting at 1s)
2. Rate limit errors (429) from Anthropic API respect the `retry-after` header
3. WebSocket clients auto-reconnect on disconnect (exponential backoff: 1s, 2s, 4s, 8s, max 30s)
4. Server-side WebSocket heartbeat (ping every 30s) detects dead connections and cleans up
5. Agent execution timeout (default: 5 min) kills stuck agents and returns partial results
6. Tool execution failure does not crash the agent — returns error message and continues
7. Conversation state survives server restart (already persisted in data/conversations/)
8. Server startup validates all dependencies (Anthropic key, PROJECT_ROOT, data dirs) and logs clear errors

## Tasks / Subtasks

- [ ] Task 1: Anthropic API retry logic (AC: 1, 2)
  - [ ] Wrap `client.messages.create()` with retry handler
  - [ ] Exponential backoff: 1s, 2s, 4s (max 3 retries)
  - [ ] On 429: use `retry-after` header value
  - [ ] On 500/502/503: retry with backoff
  - [ ] On 400/401/403: do NOT retry (client error)
  - [ ] Log each retry attempt with reason
- [ ] Task 2: WebSocket client reconnection (AC: 3)
  - [ ] Modify cockpit `app.js` WebSocket handler
  - [ ] On close/error: attempt reconnect with exponential backoff
  - [ ] Show reconnection status in UI (spinner + "Reconectando...")
  - [ ] On successful reconnect: re-authenticate and resume
  - [ ] Max reconnect attempts: 10, then show "Conexao perdida" with manual retry button
- [ ] Task 3: WebSocket server heartbeat (AC: 4)
  - [ ] Server sends `ping` every 30s to each client
  - [ ] If no `pong` within 10s, terminate connection
  - [ ] Clean up dead client from broadcast list
  - [ ] Log disconnections with reason
- [ ] Task 4: Agent execution timeout (AC: 5)
  - [ ] Wrap `executeAgent()` with AbortController/timeout
  - [ ] Default: 5 minutes per agent invocation
  - [ ] On timeout: return partial results collected so far
  - [ ] Log timeout event to audit log
  - [ ] Broadcast timeout warning to cockpit
- [ ] Task 5: Graceful tool failure (AC: 6)
  - [ ] `executeTool()` catch block returns structured error
  - [ ] Error format: `{ error: true, message: "...", tool: "...", recoverable: true }`
  - [ ] Agent receives error as tool result and can decide next action
  - [ ] Never throw from executeTool() — always return
- [ ] Task 6: Startup validation (AC: 8)
  - [ ] Check `ANTHROPIC_API_KEY` exists and format is valid
  - [ ] Check `PROJECT_ROOT` or `AIOS_ROOT` exists and is accessible
  - [ ] Check `data/` directory exists (create if missing)
  - [ ] Check `data/audit/`, `data/conversations/`, `data/learnings.json`
  - [ ] Log clear error message for each missing dependency
  - [ ] Exit with code 1 if critical dependency missing

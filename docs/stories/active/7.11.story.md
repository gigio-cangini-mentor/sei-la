## Story

**As a** AIOS developer,
**I want** unit tests for the 10+ core modules that currently lack coverage,
**so that** regressions in these modules are caught before they reach production.

## Acceptance Criteria

1. Each module below has a dedicated test file with at least basic unit tests (happy path + error case):
   - `core/permissions/index.js`
   - `core/orchestration/condition-evaluator.js`
   - `core/orchestration/skill-dispatcher.js`
   - `core/orchestration/tech-stack-detector.js`
   - `core/utils/output-formatter.js`
   - `core/utils/yaml-validator.js`
2. If `ideation-engine.js` and `timeline-manager.js` survive Story 7.10 cleanup, they also get tests
3. Each test file has at minimum 5 test cases
4. All new tests pass on first run
5. Overall test count increases by at least 50 new tests
6. No mocking of external services required (these are pure logic modules)
7. Test file naming follows convention: `tests/{module-area}/{module-name}.test.js`

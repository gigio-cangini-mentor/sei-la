## Story

**As a** JARVIS Cloud Brain operator,
**I want** tool execution sandboxed with proper isolation, timeouts, and resource limits,
**so that** agents cannot accidentally damage the system or consume excessive resources.

## Acceptance Criteria

1. All `run_command` tool executions are wrapped in a timeout (default: 60s, configurable per command type)
2. `run_command` blocks additional dangerous patterns beyond current BLOCKED_PATTERNS: `curl | bash`, backtick injection, `$(...)` in args, `eval`, `exec`
3. File write operations (`write_file`, `edit_file`) enforce a max file size (default: 1MB) and reject writes outside PROJECT_ROOT
4. `git_push` enforces branch protection: blocks force-push and push to `main` without explicit flag
5. Tool execution runs with a process-level memory limit (default: 256MB for child processes)
6. All tool executions log to an audit trail (see Story 7.3) with: tool name, agent, args, result, duration
7. A `BRAIN_SANDBOX_MODE` env var allows switching between `strict` (default) and `permissive` (for development)
8. Path traversal attacks are blocked: `..`, symlink escape, absolute paths outside PROJECT_ROOT

- [ ] Task 2: Execution timeouts (AC: 1, 5)
  - [ ] Wrap `child_process.exec` calls with configurable timeout
  - [ ] Default timeouts: `run_command` 60s, `run_tests` 300s, `git_push` 120s
  - [ ] Set `maxBuffer` to 5MB for child process output
  - [ ] Kill process tree on timeout (not just parent)
- [ ] Task 3: File operation guards (AC: 3, 8)
  - [ ] `write_file`: reject if content > 1MB
  - [ ] `write_file`/`edit_file`: resolve path and verify inside PROJECT_ROOT
  - [ ] Block writes to: `.env`, `*.key`, `*.pem`, `id_rsa*`, `docker-compose.yml`
  - [ ] Add protected paths list (configurable)
- [ ] Task 4: Git safety (AC: 4)
  - [ ] `git_push`: block `--force` and `-f` flags
  - [ ] `git_push`: block direct push to `main` (require branch)
  - [ ] `git_commit`: block `--amend` on pushed commits
- [ ] Task 5: Sandbox mode toggle (AC: 7)
  - [ ] `BRAIN_SANDBOX_MODE=strict` (default): all guards active
  - [ ] `BRAIN_SANDBOX_MODE=permissive`: log warnings but don't block
  - [ ] Log mode at startup
- [ ] Task 6: Audit logging hook (AC: 6)
  - [ ] Each tool execution emits audit event (consumed by Story 7.3)
  - [ ] Event shape: `{ timestamp, agent, tool, args, result, duration_ms, blocked }`
